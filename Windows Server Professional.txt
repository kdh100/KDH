1. 단축키
	* [Win] + [Pause] : 시스템 정보
	* [Ctrl] + E : 파일 탐색기
	* [Ctrl] + R : 실행
	* [Ctrl] + S : 검색
	* [Ctrl] + [Shift] + [ESC] : 작업 관리자
	* [Alt] + [F4] : 로그 아웃
	* [Ctrl] + [Shift] + N : 폴더 생성
	* [Win] + [Shift] + S : 화면 캡쳐
2. 명령
	* shell
		- shell:startup : 시작프로그램 폴더
		- shutdown $OPTION : 시스템 종료
			# /f : 강제종료
			# /s : 종료
			# /r : 재시작
			# /i : GUI 인터페이스 보여줌
			# /t ss : 시간 설정(default: 30s)
			# /a : 중지
			# /c : 코멘트
		- gpupdate /force : 그룹 정책 강제 업데이트
	* tool
		- control : 제어판
		- mstsc : 원격 데스크톱 연결
		- mrt : 악성 소프트웨어 제거
		- dfrgui : 디스크 조각모음
		- mspaint : 그림판
		- notepad : 메모장
		- calc : 계산기
		- taskmgr : 작업 관리자(= [Ctrl] + [Alt] + [Delete])
	* .msc(관리도구)
		- servermanager : 서버 관리자
		- msconfig : 시스템 구성
		- inetmgr : IIS 서비스 관리
		- dnsmgmt.msc : DNS 서비스 관리
		- gpedit.msc : 로컬 그룹 정책 편집기
		- lusrmgr.msc : 로컬 사용자 및 그룹
		- services.msc : 서비스
		- devmgmt.msc : 장치 관리자
		- compmgmt.msc : 컴퓨터 관리
		- diskmgmt.msc : 디스크 관리
		- fsmgmt.msc : 공유 폴더 관리
		- eventnwr.msc : 이벤트 뷰어
		- secpol.msc : 로컬 보안 정책
		- certmgr.msc : 인증서 관리
		- wbadmin.msc : 서버 백업 관리자
		- dhcpmgmt.msc : DHCP 서버 관리자
	* .cpl(제어판)
		- control userpasswords2 : 사용자 계정 접속 설정
		- firewall.cpl : 방화벽
		- ncpa.cpl : 네트워크 연결
		- inetcpl.cpl : 인터넷 속성
		- appwiz.cpl : 프로그램 및 기능

			# 복구 가능
			# 디스크 장애 시 볼륨 복구를 통해 장애난 디스크 위치에 자동 실장되어 복구

11. DHCP 서버
	* 고정 IP & 동적 IP
		- 고정은 직접 설정한 IP, 동적은 DHCP 에 의해 유동적으로 변하는 IP
			# DHCP 서버로 IP 를 사용받지 못할 경우 169.254.*.* 형식으로 IP 가 자동 할당됨
			# 인터넷 연결 불가, 내부 통신은 가능
	* DHCP 동작 방식(DORA)
		- D(discover) : C, DHCP 서버를 검색하려고 브로드캐스트 전송
		- O(offer) : S, 자신이 DHCP 서버라고 네트워크 정보와 IP 를 전송
		- R(request) : C, 네트워크 정보와 IP 사용의 허가 요청 유니캐스트 전송
		- A(ack) : S, C 의 허가 요청 승인과 네트워크 정보 임대(lease)
	* DHCP 서버 구축
		<기능 추가>
		- 서버 관리자 > 기능 추가 > DHCP 서버 체크
		<기본 설정>
		- dhcpmgmt.msc
		- IPv4 선택 > 새 범위
		- IP Pool, Default Gateway 지정
	* 임대 정보 확인
		<서버>
		- dhcpmgmt.msc
		- IPv4 > 범위 > 주소 임대 > 새로고침 > 임대받은 클라이언트 정보 확인
		<클라이언트>
		- ncpa.cpl > DHCP 로 자동 IP 할당 체크
		- cmd
		- ipconfig /all
		- ipconfig /release
			# 임대받은 IP 정보 반납
		- ipconfig /renew
			# 신규 임대 요청

13. Active Directory
	* 용어
		- 도메인(Domain) : 물리적 범주
		- 트리(Tree) : 부모-자식 도메인 계층 구조
		- 포레스트(Forest) : 두 개 이상의 트리 구성
			# 도메인 < 트리 <= 포레스트
		- 사이트(Site) : 논리적 범주
		- 트러스트(Trust) : 도메인 또는 포레스트 사이의 신뢰 관계
		- 조직 구성 단위(OU; Organizational Unit) : 도메인 내 세부적인 그룹 단위
		- 도메인 컨트롤러(DC; Domain Controller) : 로그인, 권한, 사용자 처리를 담당하는 서버
			# 도메인 서비스에서 1대 이상의 DC 가 반드시 필수
		- 읽기 전용 DC(RODC; Read-Only Domain Controller) : DC 의 부하분산 목적
		- 글로벌 카탈로그(GC; Global Catalog) : DC 가 관리해야하는 정보들의 저장 공간
	* ADDS(Active Directory Domain Service) 구축
		<구성>
		- 포레스트 : hanbit.com
			# 단일 트리

			# 부산지사(DC, GC)_SECOND

		- 도메인 선택 > 새로 만들기 > 그룹 > 내용 작성 > 그룹 종류 선택 > 생성
		<자원 공유>
		- 공유할 폴더 선택 > 속성 > 공유 > 고급 공유 > 선택한 폴더 공유 체크
		- 권한 > 기존 권한 삭제 후 추가 > 그룹이 생성된 도메인으로 위치 조정
		- 해당 자원 공유에 대한 그룹 검색 > 이름 확인 > 적용/확인
		- 클라이언트에서 공유된 자원의 경로로 네트워크 접근 확인
		<OU vs 그룹>
		- OU 는 그룹 정책의 최소 단위
		- 그룹은 사용자 관리, 권한 부여 단위
		- 사용자 계정은 1개의 OU 에만 가입 가능
		- 사용자 계정은 1개 이상의 그룹에 가입 가능
	* 그룹 정책 개체(GPO; Group Policy Object)
		<종류>
		- 로컬 GPO : 기본 컴퓨터 그룹 정책
		- 도메인 GPO : 도메인 그룹 정책
		- 사이트 GPO : 동일 도메인 내 다른 그룹 정책
			# 같은 도메인일지라도 다른 OU 로 GPO 를 관리해서 유연하게 처리 가능
		- OU GPO : 최소 단위의 그룹 정책
		<정책 전파 시간>
		- 기본값 : 90분 마다 정책 전파
		- 설정값 : 0 ~ 64800(45일)분 설정 가능
			# 0 으로 설정하면 약 7초 마다 전파 가능
			# 도메인 네트워크 트래픽이 대폭 증가되어 문제 야기
		<도메인 사용자 생성 시 암호화 관련 그룹 정책 변경>
		- 서버 관리자 > 도구 > 그룹 정책 관리
		- Default Domain Policy 선택 > 편집
		- 컴퓨터 구성 > 정책 > Windows 설정 > 보안 설정 > 계정 정책 > 암호 정책
			# 정책 수정 후 바로 적용이 안되었다면 `cmd > gpupdate /force` 진행
		<제어판 사용 제한 정책>
		- 서버 관리자 > 도구 > 그룹 정책 관리
		- 그룹 정책 개체 선택 > 새로 만들기
		- '새로 생성된 GPO' 선택 > 편집
		- 사용자 구성 > 정책 > 관리 템플릿 > 제어판
		- Prohibit access to Control Panel~ 사용 체크
		- 그룹 정책 관리 창에서 OU 선택 > 기존 GPO 연결 > '새로 생성된 GPO' 선택
			# GPO 정책이 적용된 OU 의 도메인 사용자들은 제어판을 열지 못함
		<OU 내 그룹 정책 상속 차단>
		- 서버 관리자 > 도구 > 그룹 정책 관리
		- 그룹 정책 개체 선택 > 새로 만들기
		- '새로 생성된 GPO' 선택 > 편집
		- 사용자 구성 > 정책 > 관리 템플릿 > Windows 구성 요소 > Internet Explorer
		- 홈 페이지 설정 변경할 수 없음 사용 체크
		- 그룹 정책 관리 창에서 OU 선택 > 기존 GPO 연결 > '새로 생성된 GPO' 선택
		- GPO 연결한 OU 하위의 또 다른 OU 선택 > 상속 차단
			# 강제 상속으로 상속 차단을 상쇄시킬 수 있음
			# 그룹 정책 관리 > OU 선택 > 적용된 GPO 선택 > 적용 체크
			# 다시 OU 선택 > 그룹 정책 업데이트
		<로그온 시 자동으로 프로그램 실행>
		- 서버 관리자 > 도구 > 그룹 정책 관리
		- 그룹 정책 개체 선택 > 새로 만들기
		- '새로 생성된 GPO' 선택 > 편집
		- 컴퓨터 구성 > 정책 > 관리 템플릿 > 시스템 > 로그온
		- 사용자 로그온할 때 다음 프로그램 실행 사용 체크
		- 속성 > 표시 > '\\FIRST\실습파일$\드라이브연결.bat' 입력
			# .batch 스크립트에서 @ 의 의미는 해당 라인은 재출력(echo) 하지 않겠다는 뜻
	* 그룹 정책 관리
		<백업 & 복구>
		- 서버 관리자 > 도구 > 그룹 정책 관리
		- 그룹 정책 개체 선택 > 생성된 GPO 중 백업이 필요한 정책 선택 > 백업
		- 백업 위치 지정 후 백업
		- 그룹 정책 개체 선택 > 복구
			# 바로 복구가 안될 경우 그룹 정책 관리를 다시 실행시켜 진행해볼 것
			# 다른 도메인의 그룹 정책 개체를 선택해서 GPO 를 복사할 수 있음
		<폴더 리다이렉션>
		- 공유할 폴더를 도메인 서버(FIRST)에 생성
		- '공유폴더$' 로 숨김 공유 > 'Authenticated Users' 모든 권한 부여
		- 서버 관리자 > 도구 > 그룹 정책 관리
		- 그룹 정책 개체 선택 > 새 GPO > '폴더 리다이렉션 정책' 입력/생성
		- '폴더 리다이렉션 정책' 선택 > 편집
		- 사용자 구성 > 정책 > Windows 설정 > 폴더 리디렉션 선택
		- 공유할 폴더로 경로를 재지정할 대상 위치를 선택 > 속성
		- 기본 - 모든 사용자의 폴더를~ 선택 > 루트 경로에 '\\FIRST\공유폴더$' 입력 > 확인
		- 그룹 관리 정책 > '폴더 리다이렉션 정책' 적용시킬 OU 선택 > 기존 GPO 연결 > 확인
			# 필요하다면 gpupdate /force 진행
			# 너무 많은 리다이렉션은 도메인 트래픽 부하 유발
14. 서버 보안 강화
	* Bitlocker
		- 파일, 디렉토리 단위 암호화는 EFS, 파일시스템 단위 암호화는 Bitlocker 드라이브 암호
			# 하드디스크를 암호화하여 잠그는 기능
		<구동 조건>
		- 일반적으로 TPM(Trusted Platform Module)이 메인보드에 내장되어 있어야 함
			# 소프트웨어 방식으로도 암호화 사용 가능
		- BIOS 에서 TPM 기능을 지원해야 함
		<기능 추가>
		- 서버 관리자 > 기능 추가 > Bitlocker Drive Encryption 체크 > 관리 도구 포함 체크해제
		- Bitlocker 용 HDD(2G) 추가 > 디스크 작업
		<암호화 방식 설정>
		- gpedit.msc > 컴퓨터 구성 > 관리 템플릿 > Windows 구성 요소
		- Bitlocker 드라이브 암호화 선택
		- '드라이브 암호화 방법 및 암호화 수준 선택' 사용 > AES 256 선택
		- 운영 체제 드라이브 선택
		- '운영 체제 드라이브에 드라이브 암호화 종류 적용' 사용 > 전체 암호화 선택
		- '시작 시 추가 인증 요구 선택' 사용
		- 고정 데이터 드라이브 선택
		- '고정 데이터 드라이브에 드라이브 암호화 종류 적용' 사용 > 전체 암호화 선택
		- 이동식 데이터 드라이브 선택
		- '이동식 데이터 드라이브에 드라이브 암호화 종류 적용' 사용 > 전체 암호화 선택
		- 리부팅 > administrator 로그온
		<드라이브 암호화>
		- 제어판 > 시스템 및 보안 > Bitlocker 드라이브 암호화
		- Bitlocker 용 HDD 확인 > Bitlocker 켜기 선택
		- '암호를 사용하여 드라이브 잠금 해제' 체크 > 암호 입력
		- 복구 키~ 파일에 저장 > 적당한 위치에 저장 > 암호화 시작
			# 호스트 OS 에 해당 파일 백업 받아둘 것
			# 암호 복구 시 암호 입력 칸에서 기타 옵션 > 복구 키 입력 > 48 자리 키 입력
		- 리부팅 > administrator 로그온
