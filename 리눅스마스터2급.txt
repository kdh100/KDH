[명령어] 

    * 명령행
        - PS1 : 로그인 사용자 및 호스트 도메인 정보를 알리는 프롬프트
        - PS2 : 역슬래시(\)를 쓰고 난 다음 입력을 받을 수 있는 꺽쇠(>) 프롬프트

    * 프로세스
        - nohup : 해당 프로세스를 백그라운드로 유지
        - bg : 해당 프로세스를 백그라운드로 전환
        - startx : X 윈도우 실행
        - top : CPU 사용률 확인
        - pstree : 프로세스를 트리 형식으로 확인
        - ps : 프로세스 확인
            # -aux : 시스템에서 사용중인 모든 프로세스 정보 확인
                ~ a : 실행중인 모든 프로세스
                ~ u : 사용자 이름, 프로세스 시작 시간
                ~ x : 모든 터미널에서의 정보 확인
    
    * 우선순위
        - NI 값 범위 : -20 ~ 19
            # nice <-10 | --10> bash
            # renice <10 | -10> 1222
                ~ 일반사용자는 증가 가능, 루트 사용자는 증가/감소 가능
        - ps -l : 백그라운드 프로세스의 우선순위 확인

    * 시그널
        - kill %2 : 기본값으로 15번 시그널을 2번 백그라운드 잡에 보냄
            # 1(SIGHUP) : 재시작
            # 2(SIGINT) : 프로세스 종료, Ctrl + C
            # 3(SIGQUIT) : 종료, Ctrl + \
            # 9(SIGKILL) : 강제 종료
            # 15(SIGTERM) : 시그널(정상 종료)
            # 20(SIGSTP) : 프로세스 백그라운드, Ctrl + Z

    * 스케쥴링
        - crontab <분> <시> <일> <월> <요일> <작업내용>
            # 주기로 실행시키려면 */ 을 해당 조건 숫자 앞에 붙여야함

    * 아카이브
        - zip < gzip < bzip < xz
        - tar (  ) file.tar.(  )
            # t : 내용 살펴보기
            # r : 기존 아카이브에 파일 추가(append)
            # c : 압축
            # x : 해제
                ~ J : xz
                ~ j : bz2
                ~ Z : zip(compress)
                ~ z : gz
            # -c : 파일이나 디렉토리를 묶음
            # -C : 디렉토리 변경

    * 소스 컴파일

            # make clean : 이전 컴파일 내용 삭제

[디스크]

    * 기술 
        - Mirroring : 디스크에 에러 발생 시 데이터의 손실을 막기 위해 추가로 하나 이상의 장치에 중복 저장
        - Volume Group : 물리 볼륨을 합쳐서 하나의 물리적 그룹으로 만드는 것
        - Striping : 연속된 데이터를 여러 디스크에 나눠 저장
    
    * 파일시스템
        - /etc/fstab : 파일시스템 정보를 가지고 있는 파일
            # <파일시스템 장치> <마운트 포인트> <파일시스템> <옵션> <덤프> <파일점검 옵션>
        - xfs_repair : 파일시스템 점검 및 복구
        - mkfs.xfs : XFS 파일시스템 생성
            # mkfs.ext4 : EXT4 파일시스템 생성
    
    * LVM (Logical Volume Manager)
        - PE : PV에서 나누어 사용하는 일종의 블록, 4MB로 할당
        - PV : 디스크를 LVM에서 사용할 수 있게 변환하는 작업
        - VG : PV가 모여 만들어진 그룹
        - LV : VG에서 사용자가 필요한 만큼 할당돼서 만들어지는 공간
        - LE : LV가 나누어진 일정한 크기의 블록 , PE와 1:1 대응
 
    * RAID
        - RAID-5 : 최소 3개의 저장 장치(디스크)를 필요로 하고 1개(약 33.3%)의 패리티를 사용
        - RAID-6 : 최소 4개의 저장 장치를 필요로 하고 2개의 패리티를 사용
            # 스트라이핑 방식, RAID-5 를 확장
            # 총 HDD 용량 = N - 2 

    * Quota
        - 쿼터 설정 과정
            # quotacheck --> edquota --> quotaon --> repquota
        - 디스크 쿼터 설정
            # /etc/fstab --> quota, uquota, usrquota
        - quota : 현재 사용자 쿼터 정보 확인
        - edquota : 사용자/그룹 쿼터 설정
        - setquota : 쿼터 적용
        - repquota : 쿼터 확인
        - quotaon|off : 쿼터 ON|OFF
        - quotacheck : 쿼터 설정 갱신
        - xfs_quota : 사용자 디스크 사용량 제한
